Yarner

Yarner é um projeto que tem como objetivo melhorar os hábitos culturais de leitura e escrita através de uma plataforma que permite a criação, leitura e compartilhamento de livros digitais interativos.

Virada das Coisas

A Virada Cultural é um evento que acontece anualmente em diversas cidades do Brasil. Ela reúne arte em suas mais diversas formas para todos, de forma gratuita, incluindo apresentações de circenses, dança, shows e outras atrações.

Esse projeto procura construir elementos de design baseado em Internet das Coisas (IoT) para o evento. Algumas dessas interações podem não estar disponíveis no momento, mas seriam possíveis num futuro próximo. A ideia central é usar IoT para criar mais interações sociais através de displays e outras tecnologias que estariam espalhadas pela cidade.

Mais sobre o projeto pode ser visto no GitHub

Void Call

Void Call é um jogo de tiro side scrolling.

O jogador é um herói que veio do espaço e escutou o "Chamado do Vazio" (The Void Call). Esse chamado é descrito pelas lendas como o grande sinal para saber as respostas do Universo. A aventura faz o herói cruzar diversos mundos enquanto tenta sobreviver aos desafios do "Chamado do Vazio".

A primeira versão foi desenvolvida em Java para PCs. Existe uma versão mobile em desenvolvimento.

Rafael fez todo o projeto, incluindo a narrativa do jogo, design dos personagens, arte e a codificação.

O projeto foi apresentado entre 2012 e 2014 no "Cotuca de Portas Abertas", um evento para mostrar projetos desenvolvidos nos cursos do Colégio Técnico de Campinas, da Universidade Estadual de Campinas.

Sistemas de Gerenciamento de Conteúdo

Rafael trabalhou no desenvolvimento de diferentes tipo de Sistemas de Gerenciamento de Conteúdo (do inglês, CMS).

Foi desenvolvido um CMS para um museu de artes para permitir o controle das peças disponíveis num museu. O aplicativo permitia guardar fotos e metadados sobre cada peça. O desenvolvimento foi feito em Delphi, em 2012.

Ainda em 2012, Rafael desenvolveu um CMS para uma biblioteca. Além do comportamento padrão do CMS para os livros, existiam funcionalidades para controlar os usuários da biblioteca e quais livros estavam emprestados. O desenvolvimento foi feito em Delphi.

Para poder gerenciar melhor os mapas para um jogo MOBA de rede local, Rafael desenvolveu um CMS. Ele permitia a criação de mapas usando os sprites do jogo, criação de unidades colidíveis no mapa e organizar e carregar os mapas criados no jogo. Desenvolvido em 2013, em Java.

Em 2014, Rafael desenvolveu com Edylson Torikai um CMS para artistas e músicas. A aplicação funcionava como um banco de dados com uma UI muito interativa para gerenciar as músicas. O desenvolvimento foi feito com JSP e JavaScript.

Entre 2015 e 2016, Rafael desenvolveu um CMS com vários tipos de conteúdo (vídeos, livros, jogos, personagens) que seria servido globalmente. O CMS possuía diversas regras de negócio e necessidades específicas para cada tipo de conteúdo. Ele permitia armazenar as informações em árvores-enária e criar a localização para cada conteúdo. Foi desenvolvido com Ruby on Rails e JavaScript.

Estudos em Estruturas de Dados

Com o objetivo de estudar estruturas de dados, Rafael desenvolveu alguns projetos em Java em 2013.

"Turista" foi uma aplicação que permitia o usuário criar mapas com localidades e caminhos entre esses pontos. Usando o peso de cada caminho era possível obter os caminho mais curtos entre dois pontos, o menor caminho que passasse exclusivamente por alguns pontos ou que passasse por alguns lugares. Era possível tomar notas para cada caminho, com a intenção de ser usado por turistas. As anotações poderiam ser usadas para criar uma agenda com o que seria feito em cada dia / hora e também era possível customizar os mapas.

"Lar" foi uma aplicação criada para compactar arquivos. Usando o algoritmo da Árvore de Huffman, ela permitia a compressão de arquivos no formato ".lar". Era possível colocar uma senha para proteger os arquivos.

Cliente de Email

Em 2014, Rafael desenvolveu um cliente de email usando JSP, Java e JavaScript.

O cliente web de email permitia os usuários se conectarem com diferentes contas (Gmail, Yahoo Mail, UOL) e ler os emails; marcá-los como importante; organizar as mensagens; responder, responder para todos e encaminhar emails, inclusive com um editor de HTML; e adicionar anexos de qualquer tipo, inclusive imagens inline.

O banco de dados usado foi o Microsoft SQL Server.
